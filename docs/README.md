# Docs 文档目录

本目录用于存放项目相关的技术文档和维护记录。

## 目录结构

```
docs/
└── maintenance_records/     # 维护记录文档
```

## maintenance_records/ 维护记录说明

记录了项目历次重要的维护、修复和优化工作：

### 脚本相关
- **撮合脚本使用说明.md** - 藏品撮合脚本的使用方法和定时任务配置
- **撮合脚本问题分析报告.md** - 撮合脚本"无法执行"问题的详细分析报告
- **修复定时任务配置.md** - 定时任务配置修复记录
- **后台脚本监控按钮修复说明.md** - 后台管理脚本监控按钮修复记录

### 其他维护记录
- **git_conflict_report.md** - Git冲突处理报告

## 查看建议

1. 维护人员在进行类似操作前，建议先查阅相关维护记录
2. 每次重要维护后，建议在此目录添加相应的文档记录
3. 文档命名建议格式：`主题_日期.md`（如：数据库优化_20260114.md）
