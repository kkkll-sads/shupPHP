# 资产确权与解锁逻辑说明 (Asset Migration & Unlocking)

本文档专注于说明用户如何从旧平台迁移资产（确权），并通过解锁流程激活资金。

## 1. 数据确权：旧平台资产迁移 (Migration Process)

本环节的核心目标是将用户在旧平台的资产凭证，转化为新平台的 **待激活金 (`pending_activation_gold`)**。

### 1.1 凭证提交 (Credential Submission)
*   **用户操作**: 用户需向平台提供旧平台资产持有凭证（如：旧平台账号截图、持仓记录、身份信息等）。
*   **渠道**: 
    *   *目前通常通过线下客服或专属提交入口进行（具体视运营配置）。*
    *   系统暂无自动化的 API 接口直接对接旧数据库，依赖人工或脚本审核。

### 1.2 审核与发放 (Verification & Issuance)
*   **审核**: 平台管理员核实凭证真实性。
*   **发放**: 审核通过后，管理员通过后台将等额资金划拨至用户的 **待激活金 (`pending_activation_gold`)** 账户。
*   **性质**: 待激活金属于“冻结资产”，必须通过[解锁]操作才能流转。

## 2. 旧资产解锁 (Unlocking)

用户通过特定的推广贡献，可多次申请解锁，将待激活金转化为**实物资产包**（藏品）。

### 2.1 解锁条件 (Repeatable)
用户可**多次执行**解锁，每次解锁需满足以下所有条件：
*   **身份要求**: 用户自身必须是 **交易用户**。
*   **资金要求**: `pending_activation_gold` ≥ **1000.00**。
*   **推广要求**: **每新增 3 个一级直推交易用户**，获得 1 次解锁资格。
    *   *逻辑*: 系统需记录用户当前的“有效直推数”与“已解锁次数”，计算剩余资格。

### 2.2 解锁执行 (Execution)
接口: `/api/Account/unlockOldAssets`

| 动作 | 资金/资产变动 | 说明 |
| :--- | :--- | :--- |
| **消耗** | `pending_activation_gold` **-1000** | 消耗门槛资金 |
| **发放** | **资产包/藏品** (价值1000) **+1 个** | 发放一个等值的旧资产包（藏品） |
| **赠送** | `consignment_coupon` **+1 张** | 赠送首张寄售券用于上架 |
| **记录** | 更新解锁次数记录 | 消耗一次解锁资格 |

> **关键变化**: 解锁不再直接给钱，而是给**资产**。用户必须将其**寄售卖出**才能变现。

## 3. 旧资产变现 (Cash Out)

用户解锁获得资产包后，需通过寄售流程进行变现。

### 3.1 寄售上架
用户将旧资产包提交寄售时：
*   **自动混入**: 系统自动检查当前场次相同价格分区的所有资产包，随机选择一个资产包将该寄售记录混入其中。
*   **目的**: 让旧资产包与普通资产包自然混合，避免标识明显。

### 3.2 优先出售权
在**盲盒撮合**模式下，旧资产包享有最高匹配优先级：
*   **匹配顺序**: 旧资产包 > 老用户卖家 > 系统库存。

### 3.3 出售收益 (100%分流)
根据“旧资产无利润”原则，该资产包卖出后（售价1000）：
*   **50% 本金** -> `withdrawable_money` (可提现余额，**500元**)。
*   **50% 本金** -> `score` (消费金/积分，**500分**)。
*   **无增值利润**。
